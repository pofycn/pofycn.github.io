关于原码、反码与补码的个人理解与相互联系

1、机器数的概念

一个数在计算机中的二进制表示称为这个数的机器数，机器数是带符号的，计算机中用一个数二进制的最高位表示这个数的符号，正数的符号位为0，负数的符号位为1，比如十进制的2，计算机字长为8，则2转换成二进制表示为：00000010，如果是-2，转换成二进制表示为：10000010。

2、真值的概念

因为第一位是符号位，所以机器数的形式值就不等于真正的数值，例如上面表示的二进制数10000010，其最高位1表示负，其真正的数值是-2而不是130（10000010转换成十进制等于130）。所以，为了区别，带符号位的机器数对应的真正数值称为机器数的真值。

3、原码

原码就是符号位加上真值的绝对值, 即用第一位表示符号, 其余位表示值，比如，如果是8位二进制：

[+2]原码=00000010
[-2]原码=10000010

因为第一位是符号位，所以8位二进制的取值范围是：

二进制：
11111111~01111111
转换成十进制：
-127~127

4、反码

反码的表示方法是：正数的反码是其本身，负数的反码是在其原码的基础上, 符号位不变，其余各个位取反：

[+1] = [00000001]原码 = [00000001]反码

[-1] = [10000001]原码 = [11111110]反码

可以看出来如果一个反码表示的是负数，人脑无法直观的看出来它的数值. 通常要将其转换成原码再对其进行计算。

5、补码

补码的表示方法是：正数的补码就是其本身，负数的补码是在其原码的基础上, 符号位不变, 其余各位取反, 最后+1. (即在反码的基础上+1)：

[+1] = [00000001]原码 = [00000001]反码 = [00000001]补码

[-1] = [10000001]原码 = [11111110]反码 = [11111111]补码

对于负数, 补码表示方式也是人脑无法直观看出其数值的. 通常也需要转换成原码再对其进行计算。

6、为什么要使用补码

通过上面的内容，我们可以知道计算机又三种编码方式来表示一个数，对于正数而言：

原码=反码=补码
example:
[+2]=[00000010]原码=[00000010]反码=[00000010]补码

但是对于负数而言：

[-2]=[10000010]原码=[11111101]反码=[11111110]补码

可以看出来，负数的原码、反码、补码是完全不相同的，为什么原码是更容易被人们所接受还会有反码和补码呢？

因为人脑可以根据符号位对真值部分进行对应的加减运算，但是对于计算机，要使得计算机识别符号位再对其进行加减运算只会让逻辑电路变得更加复杂，所以人们想出了让符号位也参与运算的方法，依据运算法则：减去一个正数等于加上这个正数的相反数，即：1-1=1+（-1）=0，这样对于计算机逻辑电路的设计就相当于只有加法而没有减法了，大大简化了运算逻辑的设计。

但是对于原码而言：

1-1=1+（-1）=[00000001]原码+[10000001]原码=[10000010]原码=-2

可以看出来，如果用原码表示并且让符号位也参与运算，这样计算减法得出的结果是错误的，所以计算机内部为什么不能用原码来表示一个数。

为了解决原码进行减法运算的问题，出现了反码

1-1=1+(-1)=[00000001]原码+[10000001]原码=[00000001]反码+[11111110]反码
=[11111111]反码=[10000000]原码=-0=0

发现用反码计算减法, 结果的真值部分是正确的. 而唯一的问题其实就出现在”0”这个特殊的数值上， 虽然人们理解上+0和-0是一样的，但是0带符号是没有任何意义的，而且会有[0000 0000]原和[1000 0000]原两个编码表示0。

于是补码的出现, 解决了0的符号以及两个编码的问题：

1-1 = 1 + (-1) = [00000001]原码 + [10000001]原码
= [00000001]补码 + [11111111]补码 = [00000000]补码=[00000000]原码

这样0用[0000 0000]表示, 而以前出现问题的-0则不存在了.而且可以用[1000 0000]表示-128：

(-1) + (-127) = [10000001]原码 + [11111111]原码
= [11111111]补码 + [10000001]补码 = [10000000]补码

-1-127的结果应该是-128, 在用补码运算的结果中, [1000 0000]补 就是-128，但是注意因为实际上是使用以前的-0的补码来表示-128，所以-128并没有原码和反码表示(对-128的补码表示[1000 0000]补算出来的原码是[0000 0000]原, 这是不正确的)。

使用补码, 不仅仅修复了0的符号以及存在两个编码的问题, 而且还能够多表示一个最小数，这就是为什么8位二进制, 使用原码或反码表示的范围为[-127, +127]， 而使用补码表示的范围为[-128, 127]。

因为机器使用补码, 所以对于编程中常用到的32位int类型, 可以表示范围是： [-231, 231-1] 因为第一位表示的是符号位.而使用补码表示时又可以多保存一个最小值。
